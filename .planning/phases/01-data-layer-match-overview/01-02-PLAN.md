<objective>
Add a `triggeredBy` field to track what initiated each match analysis (wave:send, profile:update, script:scatter, etc.) and persist it in the connectionAnalyses table.

Purpose: Per D-12, the admin dashboard sheet panel needs to show what triggered each analysis. Since completed jobs are removed from BullMQ (removeOnComplete: true), the trigger source must be stored in the database alongside the analysis results. This also addresses Research open question 1 by choosing option (b): persist triggeredBy in the DB.
Output: New DB column, updated job interfaces, all enqueue call sites passing triggeredBy, worker persisting it.
</objective>
